What’s it all about?

CSS is used to style the content created in an HTML document, for laying out the items and specifying how they look...if the HTML page is a person's body then the CSS is the clothing.

This article will show you the different ways you can select your HTML elements to apply the styles to...in other words how you dress up your body to look good!

Do you know?

This article won’t really make much sense unless you already have some knowledge of HTML’s semantics…and of course the actual CSS styles to apply to the element you have selected

But if you know your <p> from a <h1> and <section> from <main> then read on...

Simple Selectors

Let’s start off simple with the easiest and most basic of them all....the simple selector.

The style is applied to one or more elements based on the type of element, it’s class, or it's id. Simple eh!

Element Selector

The style is applied to every element of that type and is useful for applying page wide or fallback styles

html example:

<!--intro section-->
<p>A paragraph element is normally one of the most commonly occurring elements in any html document</p>

css example:

/*All p elements are blue*/
p {
color: blue
}

The end result is...

A paragraph element is normally one of the most commonly occurring elements in any html document

Class Selector

The class selector consists of a dot (.) followed by a class name (with no spaces in the class name).

Multiple elements in the document can have the same class name and a single element can have multiple class names (separated by spaces) which makes this selector type useful when applying a style to a variety of different types of elements.

html example:

<!--intro section-->
<section>
<p>Do I have an outline?</p>
</section>
<section class="box-outline">
<p>Or do I have an outline?</p>
</section>

css example:

/*Add a solid black border*/
.box-outline {
border: 2px solid black;
}

The end result is...

Do I have an outline?
Or do I have an outline?

In this example each element with the class "box-outline" will have this style applied to it.

ID Selector

The ID selector consists of a hashtag (#) followed by ID name (must be unique) of a given element

This is different to class styling as multiple html elements can use the same class but only a single element can have an id (There can be only one!)

html example:

<!--intro section-->
<div id="top-box">
<p>Description...</p>
</div>
<!--content section-->
<div id="not-top-box">
<p>Content...</p>
</div>

css example:

/*styling for first box in form*/
#top-box {
background-color: grey; border: 2px dashed black;
}

The end result is...

Description...
Content...

In this example it is only the element with the id "top-box" that will have this style applied to it.

Universal Selector

The most simple selector of them all may be the universal selector

* is the universal selector and is rarely useful by itself but often used in combination with other selectors.

html example:

<!--my list-->
<ul>
<li>Listy McListface</li>
<li>Listy McListyFace</li>
</ul>

css example:

/*reset default margins*/
ul * {
margin: 0; padding: 0;
border: 1px dashed grey;
}

The end result is...

With style applied:

  • Listy McListface
  • Listy McListyFace

Without style applied:

  • Listy McListface
  • Listy McListyFace

I use this is when resetting any default padding and margin on the html page itself

Attribute Selectors

Now that we have warmed up our thinking caps with the easy stuff we can start getting to the more complicated (and therefore oh so much more fun) selectors we can play around with.

Attribute selectors match one or more elements based on their attributes/attribute values.

The generic syntax consists of square brackets [] containing an attribute name followed by optional condition to match against the value of the attribute (this makes much more sense when you see the examples below).

There are two categories of attribute selectors:

  1. Presence and Value
  2. Substring Value

Presence and Value

Try to match an exact attribute or attribute value of an element. This will make much more sense when you see the examples below (Well...a bit more sense)

  • [attr] – this will select all elements with the attribute attr
  • [attr=val] – this will select all elements with the attribute attr but only if attr value is val
  • [attr~=val] – this will select all elements with the attribute attr but only if the value val is one of a space separated list of values for attr

html example:

<li>No style</li>
<!--Look #1-->
</ul>

Substring Value

The next level to presence and value, this offers flexible matching in a similar fashion to regular expression (also known as RegExp-like selectors)

  • [attr|=val] – this will select elements with the attribute attr for which the value is exactly val or starts with val
  • [attr^=val] – this will select all elements with the attribute attr for which the value starts with val
  • [attr$=val] – this will select all elements with the attribute attr for which the value ends with val
  • [attr*=val] – this will select all elements with the attribute attr for which the value contains the string val (unlike [attr~=val] it does not treat spaces as value separators but as part of attribute name)
