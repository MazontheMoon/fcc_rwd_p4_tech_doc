What is it all about?

CSS is used to style the content created in an HTML document, for laying out the items and specifying how they look...if the HTML page is a person's body then the CSS is the clothing.

This article will show you the different ways you can select your HTML elements to apply the styles to...in other words how you dress up your body to look good!

Do you know?

This article won’t really make much sense unless you already have some knowledge of HTML’s semantics…and of course the actual CSS styles to apply to the element you have selected

But if you know your <p> from a <h1> and <section> from <main> then read on...

Simple Selectors

Let’s start off simple with the easiest and most basic of them all....the simple selector.

The style is applied to one or more elements based on the type of element, it’s class, or it's id. Simple eh!

Element Selector

The style is applied to every element of that type and is useful for applying page wide or fallback styles

html example:

<!--intro section-->
<p>A paragraph element is normally one of the most commonly occurring elements in any html document</p>

css example:

/*All p elements are blue*/
p {
color: blue
}

The end result is...

A paragraph element is normally one of the most commonly occurring elements in any html document

Class Selector

The class selector consists of a dot (.) followed by a class name (with no spaces in the class name).

Multiple elements in the document can have the same class name and a single element can have multiple class names (separated by spaces) which makes this selector type useful when applying a style to a variety of different types of elements.

html example:

<!--intro section-->
<section>
<p>Do I have an outline?</p>
</section>
<section class="box-outline">
<p>Or do I have an outline?</p>
</section>

css example:

/*Add a solid black border*/
.box-outline {
border: 2px solid black;
}

The end result is...

Do I have an outline?
Or do I have an outline?

In this example each element with the class "box-outline" will have this style applied to it.

ID Selector

The ID selector consists of a hashtag (#) followed by ID name (must be unique) of a given element

This is different to class styling as multiple html elements can use the same class but only a single element can have an id (There can be only one!)

html example:

<!--intro section-->
<div id="top-box">
<p>Description...</p>
</div>
<!--content section-->
<div id="not-top-box">
<p>Content...</p>
</div>

css example:

/*styling for first box in form*/
#top-box {
background-color: grey;
border: 2px dashed black;
}

The end result is...

Description...
Content...

In this example it is only the element with the id "top-box" that will have this style applied to it.

Universal Selector

The most simple selector of them all may be the universal selector

* is the universal selector and is rarely useful by itself but often used in combination with other selectors.

html example:

<!--my list-->
<ul>
<li>Listy McListface</li>
<li>Listy McListyFace</li>
</ul>

css example:

/*reset default margins*/
ul * {
margin: 0; padding: 0;
border: 1px dashed grey;
}

The end result is...

With style applied:

  • Listy McListface
  • Listy McListyFace

Without style applied:

  • Listy McListface
  • Listy McListyFace

I use this is to reset any default padding and margin on the html page itself so any browser presets do not mess up the look of my page

Attribute Selectors

Now that we have warmed up our thinking caps with the easy stuff we can start getting to the more complicated (and therefore oh so much more fun) selectors we can play around with.

Attribute selectors match one or more elements based on their attributes/attribute values.

The generic syntax consists of square brackets [] containing an attribute name followed by optional condition to match against the value of the attribute (this makes much more sense when you see the examples below).

There are two categories of attribute selectors:

  1. Presence and Value
  2. Substring Value

Presence and Value

Try to match an exact attribute or attribute value of an element. This will make much more sense when you see the examples below (Well...a bit more sense)

  • [attr] – this will select all elements with the attribute attr
  • [attr=val] – this will select all elements with the attribute attr but only if attr value is val
  • [attr~=val] – this will select all elements with the attribute attr but only if the value val is one of a space separated list of values for attr

html example:

<!--my stylish list-->
<ul>
<li>No style</li>
<!--Look #1-->
<li class="class">Style 1</li>
<!--Look #2-->
<li class="classy">Style 2</li>
<!--Look #3-->
<li class="class classless">Style 3</li>
</ul>

css example:

/*Look #1 = style for all li elements with a class attribute*/
li[class] {
list-style: none;
}
/*Look #2 = style for li elements with a class attribute with the value "classy"*/
li[class="classy"] {
background-color: grey;
}
/*Look #3 = style for li elements with a class attribute with a value of “class” but only when “class” is not the only value for that attribute - say that five times fast*/
li[class~="class"] {
color: purple;
}

The end result is...

  • No style
  • Style 1
  • Style 2
  • Style 3

Substring Value

The next level to presence and value, this offers flexible matching in a similar fashion to regular expression (also known as RegExp-like selectors)

  • [attr|=val] – this will select elements with the attribute attr for which the value is exactly val or starts with val
  • [attr^=val] – this will select all elements with the attribute attr for which the value starts with val
  • [attr$=val] – this will select all elements with the attribute attr for which the value ends with val
  • [attr*=val] – this will select all elements with the attribute attr for which the value contains the string val (unlike [attr~=val] it does not treat spaces as value separators but as part of attribute name)

html example:

<!--my stylish list-->
<ul>
<li>No style</li>
<!--Look #4 & #5 & #6 & #7-->
<li class="class">Some style</li>
<!--Look #5 & #7-->
<li class="classy">More style</li>
<!--Look #5 & #7-->
<li class="class classless">More style but not stylish</li>
<!--Look #5 & #6 & #7-->
<li class="classy classless class">Style and more style but not stylish</li>
</ul>

css example:

/*Look #4 = style for all li elements with a class attribute but only if the value is "class"*/
li[class|="class"] {
font-size: 2rem;
}
/*Look #5 = style for li elements with a class attribute where the value begins with "class"*/
li[class^="class"] {
background-color: grey;
}
/*Look #6 = style for li elements with a class attribute where the value ends with "class"*/
li[class$="class"] {
color: purple;
}
/*Look #7 = style for li elements with a class attribute where the value includes "class" anywhere in the string*/
li[class*="class"] {
text-decoration: underline;
}

The end result is...

  • No style
  • Some style
  • More style
  • More style but not stylish
  • Style and more style but not stylish

Pseudo Classes

If you thought the last section was fun wait till you get a load of the pseudo classes!

Pseudo-classes allow you to select one or more elements that exist in a certain state such as mouse hover, or an element that is the first child of its parent in the DOM tree (like the first word in a paragraph, or the first div in an article, etc.)

A pseudo-class is a keyword preceded by a colon(:) added to the end of selectors to specify you want to style the selected elements but only when in a certain state such as mouse hover.

html example:

<!--my nav bar-->
<ul>
<!--links-->
<li>
<a>Learn</a>
</li>
...
</ul>

css example:

/*Change color of link on mouse hover*/
a:hover {
color: purple;
}

The end result is...

  • Learn

Pseudo Elements

Pseudo-elements may be used more often than pseudo-classes and are used to target elements that do not technically exist in the HTML document in that they allow you to add content to an element in the HTML document.

Similar to pseudo-classes but are preceded with two colons (::) and are added to the end of selectors to select a certain part of the element.

html example:

<!--my nav bar-->
<ul>
<!--links-->
<li>
<a href="https://www.freecodecamp.org/learn/">Learn </a>
</li>
...
</ul>

css example:

/*add arrow icon after all external http links*/
[href^=http]::after {
content: '⤴';
}

Combinators & Multiple Selectors

Combinators

Not exactly selectors themselves but ways of combining two or more selectors in useful ways for very specific selections, such as only paragraphs that come directly after headings

CSS has several ways to select elements based on how they are related to one another:

Group of selectors (A, B)
Any element matching A and /or B
Descendant selector (A B)
Any element matching B which is a descendant of an element matching A
Child selector (A > B)
Any element matching B that is a direct child of an element matching A
Adjacent sibling selector (A + B)
Any element matching B that is the next sibling of an element matching A
General sibling selector (A ~ B)
Any element matching B that is one of the next siblings of an element matching A

html example:

<!--my page-->
<h1>My Sample page</h1>
<p>This is what my page is about</p>
<!--my nav bar-->
<ul class="main-nav">
<!--links-->
<li>
<img src="logo.png">
</li>
<li>
<a href="https://www.freecodecamp.org/learn/">Learn</a>
</li>
<li>
<a href="https://www.freecodecamp.org/news/">News</a>
</li>
<li>
<a href="https://www.freecodecamp.org/forum/">Forum</a>
</li>
<li>
<!--submenu links-->
<ul class="sub-nav">
<li>...</li>
<li>...</li>
</ul>
</li>
</ul>
<p>This is the rest of my content...</p>

css example:

/*All <img> within <li> and/or all <a> within <li>*/
li img, li a {
border: 2px solid red;
}
/*Any <a> elements within a <li> element*/
li a {
border: 1px dotted black;
}
/*Any <ul> within a <li> element*/
li > ul {
background-color: grey;
}
/*Any <p> element that comes immediately after <h1>*/
h1 + p {
font-size: 1.5rem;
}
/*All <p> that come after <h1>*/
h1 ~ p {
color: darkgrey;
}

Multiple Selectors

Not separate selectors; the idea is that you can put multiple selectors on the same CSS rule, separated by commas, to apply a single set of declarations to all the elements selected by those selectors

css example:

/*fallback font style*/
h1, h2, h3, h4, h5, h6, p, li, a {
font: italic normal bold normal 1em/1.5 Helvetica, Arial, sans-serif;
}

Best Coding Practice - My rule of 3

  1. Keep it simple - start off with the basics and then evolve the style (Oh believe me, I have learned the hard way about biting off more than you can chew!)
  2. Don't repeat yourself - if all buttons have common styles use a class to apply this style and then target each button for the specific different styling
  3. Save yourself the headache of having to revisit (and revisit again) your HTML code by planning out your page first so you can add a class (or id) to your elements as you go along (and don’t forget about your container divs)...
